# PicMap 项目功能介绍

---

> 一个基于 Vue3 + Express 的本地图片地图管理应用，让你的照片在地图上展示！

---

## 核心特性

| 特性 | 说明 |
|------|------|
| 🔒 本地存储 | 所有数据保存在本地，无隐私泄露风险 |
| 🗺️ 地图展示 | 支持多种地图瓦片，直观展示照片位置 |
| 📤 批量上传 | 一次选择多张图片，支持 HEIC/HEIF 格式 |
| 👥 多用户 | 支持多用户系统，数据相互隔离 |
| 💾 数据备份 | 支持备份、导入、恢复，安全可靠 |
| 🌍 国际化 | 支持中文、英文等多种语言 |

---

## 功能模块

### 1. 图片上传与管理

- **批量上传**：支持一次性选择多张图片上传，加载过程中按钮禁用，防止重复操作
- **自动 GPS 解析**：自动从图片 EXIF 信息中提取 GPS 坐标，获取过程显示加载动画
- **手动定位**：支持为没有 GPS 信息的图片手动设置位置（点击地图选择坐标）
- **图片预览**：上传前可预览图片名称、GPS 坐标等基本信息
- **图片删除**：支持删除单张或多张图片

### 2. 地图展示

- **多种地图瓦片**：支持高德地图、百度地图、OpenStreetMap 等多种在线地图瓦片
- **瓦片管理**：在设置中可添加、删除、启用/禁用地图瓦片
- **标记点显示**：在地图上以标记点形式显示照片位置
- **图片详情**：点击标记可查看图片详情抽屉
- **视角控制**：支持缩放、平移、定位到指定图片
- **纯图模式**：可隐藏侧边栏，全屏查看地图

### 3. 图片分组

- **创建分组**：创建多个分组进行图片分类
- **分组着色**：每个分组可设置不同颜色
- **分组筛选**：按分组筛选显示地图上的图片标记
- **批量分组**：支持批量设置图片的分组
- **右键菜单**：通过右键菜单快速设置分组

### 4. 用户管理

- **多用户**：支持创建多个用户，每个用户有独立的数据空间
- **用户切换**：快速切换不同用户，查看各自的数据
- **用户删除**：删除用户及其所有数据
- **用户头像**：支持设置用户头像
- **数据隔离**：每个用户的数据相互隔离，互不影响

### 5. 数据备份与恢复

- **创建备份**：一键将所有数据打包成 ZIP 备份文件
- **备份历史**：查看所有已创建的备份文件列表
- **数据恢复**：从备份文件恢复数据
- **导入模式**：
  - 覆盖模式：清空现有数据，导入备份数据
  - 合并模式：保留现有数据，追加备份中的新数据
- **备份删除**：删除不需要的备份文件

### 6. 系统设置

- **语言切换**：支持中文、英文等多种语言
- **地图瓦片配置**：添加、编辑、删除自定义地图瓦片
- **封面图片设置**：为地图瓦片设置封面图片

## 支持的图片格式

| 格式 | 扩展名 | GPS 支持 | 备注 |
|------|--------|:--------:|------|
| JPEG | .jpg / .jpeg | ✓ | 常用照片格式 |
| PNG | .png | ✓ | 支持透明背景 |
| GIF | .gif | ✗ | 动态图片 |
| WEBP | .webp | ✓ | 现代图片格式 |
| HEIC | .heic | ✓ | iPhone 常用格式 |
| HEIF | .heif | ✓ | 高效图片格式 |
| BMP | .bmp | ✓ | 位图格式 |

> 提示：PicMap 通过读取图片 EXIF 信息中的 GPS 数据来定位照片位置。部分图片可能没有 GPS 信息，需要手动设置位置。

## 数据存储

### 目录结构

```
D:\PicMap\
├── appSchema.json          # 应用配置文件（用户列表、地图瓦片配置等）
├── PicMap_Backup\         # 备份文件目录
│   └── PicMap_Backup_*.zip
└── [用户ID]\               # 每个用户的数据目录
    └── images\
        ├── schema\
        │   └── schema.json  # 用户图片标注数据
        └── *.jpg            # 用户上传的图片文件
```

### 数据文件说明

| 文件/目录 | 说明 |
|-----------|------|
| appSchema.json | 应用级配置，包含用户列表、地图瓦片配置等 |
| [用户ID]/images/ | 用户上传的原始图片存储目录 |
| schema.json | 用户的图片标注数据（marker、group、imageInfo） |
| PicMap_Backup/ | 自动生成的备份文件存储目录 |

## 技术栈

### 前端

- **Vue 3**：渐进式前端框架
- **TypeScript**：类型安全的 JavaScript 超集
- **Vite**：新一代前端构建工具
- **Leaflet**：开源地图库
- **Element Plus**：基于 Vue 3 的 UI 组件库
- **Pinia**：轻量级状态管理库
- **ExifReader**：图片 EXIF 信息解析
- **heic2any**：HEIC 格式转换
- **vue-i18n**：国际化解决方案

### 后端

- **Node.js**：JavaScript 运行时
- **Express**：Node.js Web 框架
- **archiver**：ZIP 压缩
- **node-stream-zip**：ZIP 解压

## 注意事项

- 💡 所有数据保存在本地，建议定期进行备份
- 📍 只有包含 EXIF GPS 信息的图片才能自动定位，无 GPS 图片需手动设置
- 📂 默认存储路径为 D:\PicMap
- 🌐 除地图瓦片需要联网获取外，其他功能均可离线使用
- 👤 多用户数据相互隔离，删除用户会删除该用户所有数据

## 常见问题

**Q: 上传的图片没有在地图上显示？**

A: 检查图片是否包含 GPS 信息，没有 GPS 信息的图片需要手动设置位置。

**Q: 如何备份数据？**

A: 进入设置页面，点击"创建备份"按钮，系统会生成 ZIP 备份文件。

**Q: 如何切换用户？**

A: 点击界面右上角的用户图标，选择或创建其他用户。

**Q: HEIC 格式图片无法显示？**

A: 系统会自动将 HEIC 转换为 JPEG 格式存储，如果转换失败请确保图片完整性。
